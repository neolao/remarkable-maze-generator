# Circle maze: ring transitions restricted to evenly-spaced "portal" sectors
**Context:** With ADR 037/038's topology, any sector could carve an open connection to its inward parent or an outward child, at the maze generation algorithm's discretion. Because an inner ring's cell can have several outward children, the one that ends up open could sit anywhere within the parent's angular span — including its middle — with no visual cue tying the opening to any edge or landmark. Across a full maze this made ring-to-ring transitions look scattered and arbitrary, breaking the reading of the maze as a coherent set of nested rings.
**Decision:** Introduce "portal" sectors: a sparse, evenly-spaced subset of each ring's sectors, computed by `computePortalSectors`/`isPortalSector`, that are the only sectors allowed to carve an inward/outward connection. Non-portal sectors can only connect to their same-ring clockwise/counter-clockwise neighbors. Portals are spaced at least one full radial-thickness unit apart (matching `RADIAL_THICKNESS`, the same 1-unit scale used everywhere else in this topology), sector 0 is always a portal (keeping it aligned with the entrance/exit seam), and every ring keeps at least one portal so the maze never splits into an unreachable ring.
**Reason:** Restricting ring transitions to a small, predictable, evenly-spaced set of positions makes the maze's ring structure legible — a person tracing the maze can anticipate roughly where a ring boundary might open, instead of it being possible at any of a ring's dozens of sectors. Because every ring still has at least one portal and stays fully connected internally via clockwise/counter-clockwise moves, the maze remains one connected graph, so every generation algorithm (growing-tree, Kruskal, Wilson, Aldous-Broder) still produces a valid spanning tree covering every cell, unchanged.
**Rejected alternatives:** Leaving every sector eligible (ADR 037/038's original behavior) — this is exactly the "openings can appear anywhere" problem being fixed. Restricting only by random chance (e.g. reducing the probability a given connection opens) was considered, but does not guarantee any minimum spacing or visual predictability, just a different distribution of the same problem.
