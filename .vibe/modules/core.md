# Module: core
**Role:** Shared core carrying maze generation, PDF rendering, and the reMarkable Cloud client; consumed by `cli` and `web`. Maze generation, solution path computation, PDF rendering (reMarkable 2 page size), and reMarkable Cloud authentication are implemented; PDF upload is still a stub.
**Files:** `packages/core/src/index.ts`, `packages/core/src/maze.ts`, `packages/core/src/maze-solver.ts`, `packages/core/src/maze-pdf.ts`, `packages/core/src/remarkable-auth.ts`, `packages/core/src/remarkable-credential-store.ts`
**Exports:** `CORE_VERSION: string`, `generateMaze(options: GenerateMazeOptions): Maze`, `generateMazeBatch(options: GenerateMazeBatchOptions): Maze[]` (derives `seed + index` per maze), `solveMaze(maze: Maze): MazePosition[]`, `renderMazeToPdf(maze: Maze, options?: RenderMazeToPdfOptions): Promise<Uint8Array>` (solution can be shown via `"extra-page"` or `"overlay"`, default `"none"`), `renderMazeBatchToPdf(mazes: Maze[], options?): Promise<Uint8Array>` (one combined PDF, one maze per page), `renderMazeBatchToPdfs(mazes: Maze[], options?): Promise<Uint8Array[]>` (one separate PDF per maze), `authenticate(store, pairingCode, options?): Promise<RemarkableSession>`, `registerDevice(pairingCode, options?)`, `refreshUserToken(deviceToken, options?)` (reMarkable Cloud auth; `CredentialStore` is an interface only, no built-in implementation — see ADR 007), `REMARKABLE_2_PAGE_WIDTH_PT`, `REMARKABLE_2_PAGE_HEIGHT_PT`, types `Maze`, `Cell`, `CellWalls`, `GenerateMazeOptions`, `GenerateMazeBatchOptions`, `MazePosition`, `RenderMazeToPdfOptions`, `SolutionDisplayMode`, `CredentialStore`, `RemarkableCredentials`, `RemarkableAuthOptions`, `RemarkableSession`
**Depends on:** `pdf-lib` (external, PDF generation)
