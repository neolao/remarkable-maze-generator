# Module: core
**Role:** Shared core carrying maze generation, PDF rendering, and the reMarkable Cloud client; consumed by `cli` and `web`. Maze generation, solution path computation, PDF rendering (reMarkable 2 page size, with a visible opening at the entrance and exit — see ADR 010), and reMarkable Cloud authentication + upload are implemented on top of the `rmapi-js` library (verified against a real reMarkable account — see ADR 012, which supersedes the earlier hand-rolled protocol from ADR 007/008).
**Files:** `packages/core/src/index.ts`, `packages/core/src/maze.ts`, `packages/core/src/maze-solver.ts`, `packages/core/src/maze-pdf.ts`, `packages/core/src/remarkable-auth.ts`, `packages/core/src/remarkable-credential-store.ts`, `packages/core/src/remarkable-upload.ts`
**Exports:** `CORE_VERSION: string`, `generateMaze(options: GenerateMazeOptions): Maze`, `generateMazeBatch(options: GenerateMazeBatchOptions): Maze[]` (derives `seed + index` per maze), `solveMaze(maze: Maze): MazePosition[]`, `renderMazeToPdf(maze: Maze, options?: RenderMazeToPdfOptions): Promise<Uint8Array>` (solution can be shown via `"extra-page"` or `"overlay"`, default `"none"`), `renderMazeBatchToPdf(mazes: Maze[], options?): Promise<Uint8Array>` (one combined PDF, one maze per page), `renderMazeBatchToPdfs(mazes: Maze[], options?): Promise<Uint8Array[]>` (one separate PDF per maze), `authenticate(store, pairingCode, options?): Promise<RemarkableSession>` (`RemarkableSession` is now an opaque `rmapi-js` client instance; `CredentialStore` is an interface only, no built-in implementation — see ADR 007), `uploadPdf(session, filePath, visibleName, options): Promise<void>` (`options.readFile` is required, no default — keeps `core` free of Node-specific imports, see ADR 008), `REMARKABLE_2_PAGE_WIDTH_PT`, `REMARKABLE_2_PAGE_HEIGHT_PT`, types `Maze`, `Cell`, `CellWalls`, `GenerateMazeOptions`, `GenerateMazeBatchOptions`, `MazePosition`, `RenderMazeToPdfOptions`, `SolutionDisplayMode`, `CredentialStore`, `RemarkableCredentials`, `RemarkableAuthOptions`, `RemarkableSession`, `UploadPdfOptions`
**Depends on:** `pdf-lib` (external, PDF generation), `rmapi-js` (external, reMarkable Cloud protocol — requires Node.js 25+, see ADR 012)
