# Module: core
**Role:** Shared core carrying maze generation, PDF/image rendering, and the reMarkable Cloud client; consumed by `cli` and `web`. Maze generation (with a tunable 1–5 difficulty level controlling branch-point density via a growing-tree selection bias, see ADR 015), solution path computation, PDF rendering (reMarkable 2 page size, with a visible opening at the entrance and exit — see ADR 010), SVG image rendering for lightweight previews (see ADR 019), and reMarkable Cloud authentication + upload are implemented on top of the `rmapi-js` library (verified against a real reMarkable account — see ADR 012, which supersedes the earlier hand-rolled protocol from ADR 007/008).
**Files:** `packages/core/src/index.ts`, `packages/core/src/maze.ts`, `packages/core/src/maze-solver.ts`, `packages/core/src/maze-layout.ts`, `packages/core/src/maze-pdf.ts`, `packages/core/src/maze-svg.ts`, `packages/core/src/remarkable-auth.ts`, `packages/core/src/remarkable-credential-store.ts`, `packages/core/src/remarkable-upload.ts`
**Exports:** `CORE_VERSION: string`, `generateMaze(options: GenerateMazeOptions): Maze` (`difficulty` is optional, integer 1–5, defaults to 1 — see ADR 015; the returned `Maze` carries its own resolved `type`, `seed`, and `difficulty` — see ADR 016), `generateMazeBatch(options: GenerateMazeBatchOptions): Maze[]` (derives `seed + index` per maze, forwards `difficulty` to every maze, each returned maze keeps its own resolved seed), `solveMaze(maze: Maze): MazePosition[]`, `computeWallSegments(maze: Maze): LineSegment[]` (internal-facing geometry shared by every maze renderer — unit cell coordinates, encodes the entrance/exit opening rules once so PDF and SVG rendering can't visually drift apart), `renderMazeToPdf(maze: Maze, options?: RenderMazeToPdfOptions): Promise<Uint8Array>` (solution can be shown via `"extra-page"` or `"overlay"`, default `"none"`; prints a small footer with the maze's type/dimensions/seed/difficulty when `maze.seed` is set — see ADR 016), `renderMazeBatchToPdf(mazes: Maze[], options?): Promise<Uint8Array>` (one combined PDF, one maze per page), `renderMazeBatchToPdfs(mazes: Maze[], options?): Promise<Uint8Array[]>` (one separate PDF per maze), `renderMazeToSvg(maze: Maze, options?: RenderMazeToSvgOptions): string` (self-contained SVG markup, walls only, no solution overlay — used for the web preview so it renders in any browser without depending on a PDF viewer, see ADR 019), `SOLUTION_MODES: SolutionDisplayMode[]`, `isValidSolutionMode(value: string): value is SolutionDisplayMode`, `invalidSolutionModeMessage(value: string): string` (solution-mode validation shared by `cli` and `web` so both interfaces enforce and describe the same allowed values — see ADR 021), `authenticate(store, pairingCode, options?): Promise<RemarkableSession>` (`RemarkableSession` is now an opaque `rmapi-js` client instance; `CredentialStore` is an interface only, no built-in implementation — see ADR 007), `uploadPdf(session, filePath, visibleName, options): Promise<void>` (`options.readFile` is required, no default — keeps `core` free of Node-specific imports, see ADR 008; `options.folder` optionally targets an existing reMarkable Cloud folder by name — throws a clear error if it doesn't exist, no auto-create, see ADR 013), `REMARKABLE_2_PAGE_WIDTH_PT`, `REMARKABLE_2_PAGE_HEIGHT_PT`, types `Maze`, `Cell`, `CellWalls`, `GenerateMazeOptions`, `GenerateMazeBatchOptions`, `MazePosition`, `RenderMazeToPdfOptions`, `SolutionDisplayMode`, `RenderMazeToSvgOptions`, `CredentialStore`, `RemarkableCredentials`, `RemarkableAuthOptions`, `RemarkableSession`, `UploadPdfOptions`
**Depends on:** `pdf-lib` (external, PDF generation), `rmapi-js` (external, reMarkable Cloud protocol — requires Node.js 25+, see ADR 012)
