# Module: cli
**Role:** Command-line interface (Commander) generating mazes as PDF and sending them to reMarkable. The `generate` (width/height/seed/difficulty/type/algorithm/solution/path-length/path-length-candidates/output), `send` (upload a local PDF, with interactive first-time pairing), and `generate-and-send` (combines both in one call, same generation options as `generate`) commands are implemented.
**Files:** `packages/cli/src/index.ts`, `packages/cli/src/generate.ts`, `packages/cli/src/generate-and-send.ts`, `packages/cli/src/cli-options.ts`, `packages/cli/src/send.ts`, `packages/cli/src/credential-store.ts`
**Exports:** `remarkable-maze` binary (entry point `program.parse()`, subcommands `generate`, `send`, and `generate-and-send`; `generate` and `generate-and-send` support `--difficulty <number>` (1–5, see ADR 015), `--type <type>` (`rectangle`, `rectangle-crossing`, `circle`, or `circle-crossing`, defaults to `rectangle`, validated via `core`'s shared `isValidMazeType`/`invalidMazeTypeMessage` — see ADR 022, ADR 037, ADR 055), `--algorithm <algorithm>` (`growing-tree`, `kruskal`, `wilson`, or `aldous-broder`, defaults to `growing-tree`, validated via `core`'s shared `isValidMazeAlgorithm`/`invalidMazeAlgorithmMessage` — see ADR 033), `--solution <mode>` (`none`, `extra-page`, or `overlay`, defaults to `none`, validated via `core`'s shared `isValidSolutionMode`/`invalidSolutionModeMessage` — see ADR 021), `--path-length <target>` (`short`, `medium`, or `long`, defaults to unset — no path-length filtering — validated via `core`'s shared `isValidPathLengthTarget`/`invalidPathLengthTargetMessage` — see ADR 046), and `--path-length-candidates <number>` (integer, overrides `core`'s default candidate count when `--path-length` is set, parsed via `parseIntegerOption` and left to `core`'s own `generateMaze` to reject when non-positive, above the cap, or used without `--path-length` — see ADR 047; `send` and `generate-and-send` support `--folder <name>`), `runGenerate(options: GenerateOptions): Promise<GenerateResult>` (`options.solution`, `options.type`, `options.algorithm` and `options.pathLength` are raw strings validated internally, rejecting an unsupported value — or an unsupported type/algorithm combination — before generation starts; `options.pathLengthCandidateCount` is forwarded as-is to `core`'s `generateMaze`, which owns its validation), `parseIntegerOption(value, flagName): number`, `runSend(options: SendOptions): Promise<SendResult>` (prompts interactively for a pairing code on first use, per ADR 011), `runGenerateAndSend(options: GenerateAndSendOptions): Promise<GenerateAndSendResult>` (defaults the visible name to `rectangle-<width>x<height>-<seed>` per ADR 014; forwards `options.solution`, `options.type`, `options.algorithm`, `options.pathLength` and `options.pathLengthCandidateCount` to generation, rejecting any of them before any authentication or upload attempt; throws `UploadAfterGenerateError` — keeping the local PDF — if generation succeeds but upload fails), `createFileCredentialStore(filePath): CredentialStore` (JSON file at `~/.config/remarkable-maze-generator/credentials.json` by default, mode `0600`)
