# Module: web
**Role:** Web server (Fastify) serving the static web UI and a JSON/PDF/SVG API; the static page lets a user configure a maze, see an image preview, and download the matching PDF (upload-to-reMarkable from the web UI not yet exposed — see backlog 014).
**Files:** `packages/web/src/server.ts`, `packages/web/src/maze-routes.ts`, `packages/web/src/maze-form-validation.ts`, `packages/web/public/index.html`, `packages/web/public/app.js`
**Exports:** `buildServer(): FastifyInstance` — `GET /api/version`, `POST /api/mazes/generate` (accepts `width`/`height`/`seed`/`difficulty`/`solution` in the JSON body, returns the generated PDF directly with `Content-Type: application/pdf`, or a 400 JSON `{ error }` on invalid parameters or an invalid `solution` value — see ADR 017), `POST /api/mazes/preview` (accepts the same `width`/`height`/`seed`/`difficulty`, returns an SVG image with `Content-Type: image/svg+xml`, or a 400 JSON `{ error }` — shares its maze-building step with `/api/mazes/generate` to avoid duplicating request parsing, see ADR 019); `validateMazeFormInput(input): MazeFormValidationResult` (tested rules for width/height as positive integers and difficulty 1–5, mirrored by plain JS in `public/app.js` since the static page has no build step — see ADR 018); static files served from `public/`, including a maze configuration form that shows the SVG preview inline and a "Download PDF" link (shown only after a successful generation) that fetches `/api/mazes/generate` and triggers a browser download — see ADR 019 (supersedes the PDF-in-iframe preview from ADR 018)
**Depends on:** [`modules/core.md`](core.md)
