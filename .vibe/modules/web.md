# Module: web
**Role:** Web server (Fastify) serving the static web UI and a JSON/PDF API; the static page lets a user configure and preview a maze in the browser (download and upload-to-reMarkable from the web UI not yet exposed — see backlog 013/014).
**Files:** `packages/web/src/server.ts`, `packages/web/src/maze-routes.ts`, `packages/web/src/maze-form-validation.ts`, `packages/web/public/index.html`, `packages/web/public/app.js`
**Exports:** `buildServer(): FastifyInstance` — `GET /api/version`, `POST /api/mazes/generate` (accepts `width`/`height`/`seed`/`difficulty`/`solution` in the JSON body, returns the generated PDF directly with `Content-Type: application/pdf`, or a 400 JSON `{ error }` on invalid parameters or an invalid `solution` value — see ADR 017); `validateMazeFormInput(input): MazeFormValidationResult` (tested rules for width/height as positive integers and difficulty 1–5, mirrored by plain JS in `public/app.js` since the static page has no build step — see ADR 018); static files served from `public/`, including a maze configuration form that renders the generated PDF as an in-page preview via the browser's native PDF viewer
**Depends on:** [`modules/core.md`](core.md)
