## Modules
- [`modules/core.md`](modules/core.md) — shared core (maze generation pluggable across 4 selectable algorithms — growing-tree (default, difficulty-tunable, only one supporting bridge crossings), kruskal, wilson, aldous-broder — each isolated in its own module, and a choice of maze type — classic thin-wall `rectangle` or thick curvy-path `rectangle-crossing`, solution path and branch-point detection, PDF and SVG rendering sharing common wall/path-geometry layers, SVG preview with an optional solution/branch-point overlay, batch PDF generation, generation-parameters footer, shared solution-mode/maze-type/algorithm validation, reMarkable Cloud auth + upload via `rmapi-js`, verified against a real account)
- [`modules/cli.md`](modules/cli.md) — CLI (Commander) consuming `core` — `generate`, `send`, and combined `generate-and-send` commands implemented, with `--solution`, `--type` and `--algorithm` options on `generate`/`generate-and-send`
- [`modules/web.md`](modules/web.md) — web server (Fastify) consuming `core` — a maze configuration form (including a maze type dropdown, a generation algorithm dropdown, a solution display mode dropdown for the PDF, and a "show solution on preview" checkbox with a branch-point count display) with SVG image preview, PDF download, and direct send-to-reMarkable (with guided pairing and an optional target reMarkable Cloud folder) all implemented, with a dedicated responsive stylesheet applied to the page

## Observed patterns
- npm workspaces monorepo (`packages/*`), one `package.json` per package with `private: true`
- `core` exposes its TypeScript sources directly (`main`/`exports` point to `src/index.ts`, no build required for dev/test)
- A Vitest test file next to each source module (`*.test.ts`)
- Style and lint centralized via Biome at the root (`biome.json`), tab formatting, double quotes
- Requires Node.js 25+ (pinned in `.nvmrc`, declared in the root `package.json` `engines` field) — needed by `rmapi-js`'s use of native `Uint8Array` base64/hex methods
